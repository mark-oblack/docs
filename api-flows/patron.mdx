This API flow describes how a patron browses a restaurant menu and places an order.

## 1. Browse the menu

The patron uses the food ordering app to browse the menu and view dishes belonging to specific categories, such as Appetizer, Main Course, Dessert, and Drinks.

### Request

The client sends an HTTP request to retrieve dishes belonging to a specific category `Appetizer` in this example).

```shellscript
curl -L -X GET '{baseUrl}/dishes?filter=category.eq~Appetizer' \
-H 'Accept: application/json'
```

If the request was successful, the server responds with a `200 OK` status code with the response body containing the list of dishes that belong to the specified category.

## 2. Check ingredient quantities

Before displaying available dishes to the patron, the client app checks ingredient inventory. If a dish has essential ingredients whose quantity is zero, the dish will not be displayed as available for selection in the UI.

### Request

For each ingredient included in a dish, the client sends an HTTP request to retrieve the ingredient quantity.

```shellscript wrap
curl -L -X GET '{baseUrl}/ingredients/{ingredientId}?fields=in_stock_qty' \
-H 'Accept: application/json'
```

### Response

If the request was successful, the server responds with a `200 OK` status code with the response body containing the ingredient quantity for the specified ingredient.

```json
{
    "id": "698b8810a91dc520146ece6c",
    "in_stock_qty": 17
}
```

## 3. View dish details

The patron selects a dish to view more information about it.

### Request

The client sends an HTTP request to retrieve details about a specific dish, including the dish's name, price, image name, and ingredients.

```shellscript wrap
curl -L -X GET '{baseUrl}/dishes/{dishId}?fields=name%7Cprice%7Cimage_name%7Cingredients' \
-H 'Accept: application/json'
```

### Response

If the request was successful, the server responds with a `200 OK` status code with the response body containing the dish's details.

## 4. Add dishes to order

If the patron likes the dish they selected, they can add it to their order. The client keeps track of each dish the patron adds to their order.

<Note>
  Note

  When the patron views their bill, the client can retrieve details about each dish included in the order from local storage.
</Note>

## 5. Placing an order

After the patron selects all dishes they want to order, they place their order.

### Request

The client sends an HTTP request to create an order, retrieving the dishes that the patron selected from local storage and including them in the request body.

```shellscript wrap
curl -L -X POST '{baseUrl}/orders' \
-H 'Content-Type: application/json' \
-H 'Accept: application/json' \
-H 'Authorization: Bearer {accessTokenValue}' \
-d '{
  "name": "Benjamin",
  "table_number": 18,
  "dish_ids": [
    "698b8810a91dc520146ece94"
  ],
  "special_requests": "No peanuts",
  "scheduled_at": null
}'
```

### Response

If the request was successful, the server responds with a `201 Created` status code with the response body containing details about the order.

```json
{
    "id": "69bd9fcfd6cd65b962ab3ff6",
    "created_at": "2026-03-20T19:28:15.556Z",
    "updated_at": "2026-03-20T19:28:15.556Z",
    "scheduled_at": null,
    "table_number": 18,
    "status": "Received",
    "priority": 3,
    "dish_ids": [
        "698b8810a91dc520146ece94"
    ],
    "special_requests": "No peanuts"
}
```

## 6. Receive order confirmation

After the patron places an order, they receive confirmation and the order details are sent to the restaurant staff to begin preparation.
